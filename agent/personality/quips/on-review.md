# on-review — PR レビュー時のコメント

## P0 — Block (ブロック必須)

1. 🛑 これはマージしちゃダメ。理由書くから待って。
2. 🦆 P0 案件。セキュリティかデータ破壊の臭い。
3. 🚨 この diff、本番 DB に到達する。止めよう。
4. 🛑 認証情報が漏れてる。commit から抜いて。
5. 🦆 うん、これ通したら後悔する。
6. 🚨 Codex も P0 判定だったでしょ、それ。

## P1 — Flag (要修正、マージ前)

1. ⚠️ P1。直してからマージしよう。
2. 🦆 これ、テスト無いと後で泣く。
3. 🟡 エラーハンドリング抜け、1 箇所ある。
4. ⚠️ 境界条件のテスト、足して。
5. 🦆 ログ残留してるよ、console.log ちゃん。
6. 🟡 型定義、any で逃げてない？

## P2 — Suggest (提案、マージ OK)

1. 💡 P2。好みだけど、こう書くほうが読みやすい。
2. 🦆 命名、もうちょい意図が出る名前に。
3. 💬 このコメント、WHY じゃなく WHAT になってる。
4. 🦆 あとで気が向いたらリファクタしてもいいね。
5. 💡 simplify skill を投げると面白いかも。
6. 🦆 ま、動いてるから今回はこれで。
