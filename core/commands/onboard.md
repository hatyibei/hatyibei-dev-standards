コードベースの構造と設計を理解するための探索を行う。

## 手順

### 1. 基本情報

- CLAUDE.md / README.md を読む
- `package.json` (or equivalent) で技術スタックを確認
- ディレクトリ構造を把握

### 2. アーキテクチャ把握

- エントリーポイントを特定
- 主要モジュール間の依存関係をトレース
- データフローを追跡（リクエスト → レスポンス）
- 外部サービス連携を特定

### 3. 開発環境

- ビルドコマンド
- テストコマンド
- 環境変数の必要性
- CI/CD パイプライン

### 4. コード規約

- 使用されているパターン（MVC, hooks, etc.）
- テストの書き方
- 命名規則
- エラーハンドリングパターン

## 出力

```
## Codebase Overview: [プロジェクト名]

### 技術スタック
- Language: ...
- Framework: ...
- DB: ...
- Infra: ...

### 構造
[ディレクトリツリーと各ディレクトリの責務]

### エントリーポイント
- [ファイル]: [説明]

### データフロー
1. ...

### 開発コマンド
- Build: `...`
- Test: `...`
- Dev: `...`

### 知っておくべきこと
- [プロジェクト固有の注意点]
```
