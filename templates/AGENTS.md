## Review guidelines

### P0 — Must fix（マージブロッカー）

- テストなしの機能追加
- `--no-verify` / `--no-gpg-sign` の痕跡
- 認証情報のハードコード（API キー、シークレット、パスワード）
- 認証が必要な API ルートの保護漏れ
- XSS / SQL インジェクション / プロンプトインジェクション
- `console.log` に個人情報・認証情報を出力
- `git push --force` to main/master
- ビルドが通らない変更

### P1 — Should fix（強く推奨）

- Conventional Commits 形式でないコミットメッセージ
- `console.log` / `console.debug` / `console.info` の残留
- 1 PR に複数の論理的変更が混在
- linter / formatter 設定の不要な緩和
- 外部 API 呼び出しのエラーハンドリング欠落
- `any` 型の使用
- テストカバレッジの著しい低下

### P2 — Nice to have（改善提案）

- 命名規則の不統一
- 不要なコメントやデッドコード
- インポート順序の乱れ
