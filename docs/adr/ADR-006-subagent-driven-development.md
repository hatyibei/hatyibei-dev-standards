# ADR-006: サブエージェント駆動開発

## ステータス
accepted

## コンテキスト

複雑なタスクを単一のエージェントコンテキストで処理すると、コンテキストウィンドウの圧迫と注意力の分散が発生する。タスクを分割して専門化されたサブエージェントに委譲する戦略が有効。

## 決定

Superpowers のサブエージェント駆動開発パターンを採用する。

### アーキテクチャ

```
メインエージェント (オーケストレーター)
  ├── 計画サブエージェント (Plan)
  ├── 実装サブエージェント (General) × N (並列可)
  ├── コードレビューサブエージェント (Explore)
  └── テストサブエージェント (General)
```

### 原則

1. **フレッシュコンテキスト**: 各サブエージェントは新鮮なコンテキストで起動
2. **最小コンテキスト注入**: タスクに必要な情報だけを渡す
3. **並列実行**: 独立したタスクは並列でサブエージェントに委譲
4. **2段階レビュー**: 仕様準拠確認 → コード品質確認の順で実施
5. **理解の委譲禁止**: 「調査結果に基づいて修正して」ではなく、具体的な指示を与える

### タスク分解の基準

- 各タスクは **2-5分** で完了可能なサイズ
- タスク間の依存関係を明示（blocks / blockedBy）
- 各タスクに **完了条件** と **検証方法** を含める

### サブエージェントの使い分け

| タイプ | 用途 |
|-------|------|
| Explore | コードベース調査、ファイル検索 |
| Plan | 実装計画の設計 |
| General | 実装、テスト実行、複雑なタスク |
| Worktree | 隔離環境での実験的変更 |

## 根拠

- **obra/superpowers**: subagent-driven-development スキル、2段階レビュー
- **affaan-m/everything-claude-code**: 38専門エージェント、エージェントファースト委譲
- **awslabs/aidlc-workflows**: ユニット単位の並列構築

## 影響

- コンテキスト効率の大幅な向上
- タスク間のコミュニケーションオーバーヘッド
- サブエージェントへの指示品質がアウトプット品質を決定する
