# ADR-005: フック駆動自動化

## ステータス
accepted

## コンテキスト

Claude Code のフックシステムは、セッション開始・ツール使用前後・停止時にカスタムスクリプトを実行できる。これを活用して開発ワークフローを自動化したい。

## 決定

ライフサイクルフックを活用し、手動操作を最小化する自動化パイプラインを構築する。

### フックポイント

```
SessionStart       → 環境セットアップ、コンテキスト注入
UserPromptSubmit   → 入力バリデーション、セッション初期化
PreToolUse         → ツール実行前チェック（安全性検証）
PostToolUse        → 観測キャプチャ、自動フォーマット
Stop               → セッション要約、メモリ保存
SessionEnd         → クリーンアップ
```

### 実装ルール

1. **グレースフルデグラデーション**: フック失敗時は exit 0 で処理続行（ブロッキングエラーのみ exit 2）
2. **200行制限**: 各フックスクリプトは200行以内
3. **べき等性**: 同じフックを複数回実行しても安全
4. **タイムアウト設定**: 各フックに適切なタイムアウトを設定（デフォルト60秒）
5. **プロファイルによる切り替え**: 環境変数でフックの有効/無効を制御

### 推奨フック構成

| フック | 目的 | タイムアウト |
|-------|------|------------|
| session-start | Superpowersコンテキスト注入 | 60s |
| post-tool-use (Edit) | 自動フォーマット | 30s |
| post-tool-use (Bash) | セキュリティ監視 | 30s |
| stop | セッション要約 & メモリ保存 | 120s |

### 設定形式

```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [{ "type": "command", "command": "bash hooks/session-start.sh" }]
      }
    ]
  }
}
```

## 根拠

- **affaan-m/everything-claude-code**: 包括的なフックシステム、run-with-flags.js、プロファイル制御
- **thedotmack/claude-mem**: 5段階ライフサイクルフック、観測キャプチャ、要約生成
- **obra/superpowers**: session-start フックによるコンテキスト注入

## 影響

- 手動操作の大幅な削減
- フックのデバッグが難しくなる可能性（ログ出力を徹底）
- フック間の依存関係管理が必要
