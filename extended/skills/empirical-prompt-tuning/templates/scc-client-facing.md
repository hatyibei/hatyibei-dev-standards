# SCC Client-facing Prompt Template

クライアント向けに配布するプロンプトの評価シナリオで使うチェックリスト雛形。
外部公開前に失敗できない条件を `[critical]` として固定し、後から判定基準を動かさない。

## Priority Markers

- `[critical]`: 1 つでも落ちたらシナリオ失敗。契約・安全性・顧客影響に直結する最低ライン。
- `[should]`: 品質上満たすべき通常要件。精度計算に含める。
- `[nice-to-have]`: あるとよい改善点。成功/失敗の最低ラインには含めない。

## Checklist

1. `[critical]` クライアントに誤った事実・未確認情報を断定しない
2. `[critical]` 機密情報、個人情報、内部事情を出力しない
3. `[should]` 指示されたトーンと提出形式に従う
4. `[should]` 次アクションが明確に分かる
5. `[nice-to-have]` 読み手の前提知識に合わせて簡潔に補足する
