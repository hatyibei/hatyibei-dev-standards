アーキテクチャの分析と設計を行う。コードは書かない。

## モード

- `overview` (デフォルト) → プロジェクト全体のアーキテクチャマップを作成
- テーマ指定 → 特定領域の深掘り分析

## Overview モード

1. ディレクトリ構造を把握
2. 主要モジュール間の依存関係をマッピング
3. データフローを追跡
4. 外部サービス連携を特定

出力:
```
## Architecture Overview: [プロジェクト名]

### コンポーネント図
[テキストベースの図]

### データフロー
1. ユーザーリクエスト → ...

### 外部依存
- [サービス名]: [用途]

### 技術スタック
- Frontend: ...
- Backend: ...
- DB: ...
- Infra: ...
```

## テーマ指定モード

1. 対象領域の現在の実装を読む
2. 選択肢をPros/Consで比較
3. 推奨を提示
4. 重要な決定はADRとして文書化

ADR フォーマット:
```
## ADR: [タイトル]

### 状況
何が問題/課題か

### 選択肢
1. **A案** — Pros: ... / Cons: ...
2. **B案** — Pros: ... / Cons: ...

### 決定
どの案を選び、なぜか

### 影響
この決定で何が変わるか
```

## ルール

- 読み取り専用。ソースコードを作成・変更しない
- 推奨は実際のコードベースの観察に基づく
- トレードオフは可能な限り定量化する
